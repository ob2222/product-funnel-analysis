Аналіз продуктової воронки та пошук аномалій (Product Funnel Analysis) 📊

📌 Про проєкт

Цей проєкт присвячений аналізу шляху користувача (User Journey) в застосунку: від моменту відкриття до успішної оплати. Головна мета — побудувати продуктову воронку, розрахувати конверсію на кожному кроці та знайти "вузькі місця" (bottlenecks), через які бізнес втрачає клієнтів і гроші.

🛠️ Стек технологій

Python (pandas, numpy): Генерація датасету сирих логів подій (понад 40 000 подій).

SQL (Google BigQuery): Обробка та агрегація даних. Використання умовної агрегації (COUNT DISTINCT + CASE WHEN) для трансформації рядків подій у колонки кроків воронки.

Tableau Public: Візуалізація даних, застосування функції Pivot для розгортання таблиці, побудова Funnel Chart та графіків аномалій.

📂 Процес та методологія

Підготовка даних: Згенеровано сирий лог подій з таймстемпами для 3 платформ (iOS, Android, Web).

Трансформація (SQL): Сирі кліки згруповано за платформами. Розраховано кількість унікальних користувачів, які дійшли до кожного етапу: app_open -> view_catalog -> add_to_cart -> checkout_start -> payment_success.

Візуалізація: Побудовано дашборд для порівняння конверсії між платформами та аналізу помилок на етапі оплати.

💡 Ключові бізнес-висновки та рекомендації

🚨 Критична аномалія на Android: Аналіз воронки виявив катастрофічне падіння конверсії на фінальному етапі (payment_success) саме серед користувачів Android.

🔍 Підтвердження проблеми: Аналіз метрики payment_errors показав, що рівень помилок оплати на Android складає близько 70%, тоді як на iOS та Web цей показник знаходиться в межах норми (10-15%).

💰 Бізнес-імпакт: Через цей баг компанія щодня втрачає готові замовлення та маркетингові бюджети, витрачені на залучення цих клієнтів.

🎯 Рекомендація: Негайний ескалаційний тікет (High Priority) команді розробки Android та QA для тестування екрану оплати та інтеграції з платіжним шлюзом. До виправлення бага рекомендується тимчасово перерозподілити маркетинговий бюджет на iOS та Web-кампанії.
